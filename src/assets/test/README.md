# 测试数据文件说明

本目录包含用于测试单词导入功能的 JSON 数据文件。

## 文件列表

### 1. `test_words.json`
**完整测试数据集**（推荐用于全面测试）

- **单词数量**: 15 个
- **特点**:
  - 包含不同难度系数（1-5）
  - 包含多个单词集：Default、日常用语、动词、形容词、学术词汇
  - 包含有备注和无备注的单词
  - 涵盖常用日语单词

**使用场景**: 测试完整的导入流程，包括单词集自动创建功能

### 2. `test_words_minimal.json`
**最小测试数据集**（用于快速测试）

- **单词数量**: 2 个
- **特点**:
  - 最简单的数据结构
  - 包含有 wordSet 和无 wordSet 的示例
  - 适合快速验证基本导入功能

**使用场景**: 快速测试导入功能是否正常工作

### 3. `test_words_multiple_sets.json`
**多单词集测试数据集**（用于测试单词集功能）

- **单词数量**: 7 个
- **特点**:
  - 包含多个不同的单词集：问候语、动词、形容词
  - 测试单词集自动创建功能
  - 每个单词集包含多个单词

**使用场景**: 专门测试单词集相关的导入功能

## 数据格式说明

所有 JSON 文件都遵循以下格式：

```json
[
  {
    "kana": "假名（必填）",
    "kanji": "汉字（必填）",
    "meaning": "意思（必填）",
    "example": "例句（必填）",
    "mark": "备注（可选，可为空字符串）",
    "difficultyCoefficient": "难度系数（1-5，字符串格式）",
    "wordSet": "单词集名称（可选，不提供则使用默认词集）"
  }
]
```

## 字段说明

- **kana**: 假名，必填字段，不能为空
- **kanji**: 汉字，必填字段，不能为空
- **meaning**: 意思，必填字段，不能为空
- **example**: 例句，必填字段，不能为空
- **mark**: 备注，可选字段，可以为空字符串
- **difficultyCoefficient**: 难度系数，必须是 "1" 到 "5" 之间的字符串
- **wordSet**: 单词集名称，可选字段
  - 如果提供且不为空，系统会自动创建该单词集（如果不存在）
  - 如果值为 "Default"，会使用默认单词集
  - 如果不提供或为空，会使用默认单词集

## 使用方法

1. 在应用中打开导入对话框
2. 选择文件格式为 "JSON"
3. 选择编码为 "UTF-8"
4. 点击"选择文件"按钮
5. 导航到 `src/assets/test/` 目录
6. 选择任意一个测试文件
7. 点击"导入"按钮

## 注意事项

- 所有文件都使用 UTF-8 编码
- 文件大小都在安全限制范围内（< 10MB）
- 所有数据都符合导入验证规则
- 重复导入会创建重复的单词记录
