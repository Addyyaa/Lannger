# 工单系统说明

## 📋 工单系统使用指南

本目录用于管理所有开发任务，采用工单（Ticket）方式进行记录和跟踪。

### 工单格式

每个工单文件命名格式：`TICKET-{编号}.md`

工单文件包含以下信息：

- **工单编号**：唯一标识符（如 TICKET-001）
- **创建时间**：时间戳（精确到分钟）
- **创建者**：谁提出的工单
- **分配给**：谁负责处理
- **状态**：待处理 / 进行中 / 已完成 / 已取消
- **优先级**：P0（紧急）/ P1（重要）/ P2（一般）
- **详细内容**：工单的具体描述和要求

### 工单状态说明

- **待处理**：工单已创建，等待处理
- **进行中**：正在处理中
- **已完成**：已完成并验收
- **已取消**：不再需要处理

### 如何创建工单

1. 在 `todo.md` 中查看工单列表
2. 创建对应的工单文件（`tickets/TICKET-{编号}.md`）
3. 更新 `todo.md` 中的工单状态

### 如何查看工单

- 查看 `todo.md` 获取工单列表和状态
- 查看 `tickets/` 目录下的具体工单文件获取详细信息
